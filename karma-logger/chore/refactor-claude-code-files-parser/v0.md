## Summary

Extract the JSONL parser from `karma-logger` into a standalone package `claude-code-files-parser/`. This creates a reusable library for parsing Claude Code session logs, independent of karma-logger's aggregation and dashboard concerns.

## Motivation

1. **Single Responsibility**: Parser knows "Claude Code file format", not "karma metrics"
2. **Reusability**: Other tools can parse Claude logs without karma-logger dependency
3. **Cleaner Architecture**: karma-logger becomes a consumer, not owner of format knowledge
4. **Testability**: Parser can be tested in isolation with mock JSONL files

## Workspace Configuration

Add to root `package.json`:

```json
{
  "workspaces": [
    "karma-logger",
    "claude-code-files-parser"
  ]
}
```

Or use npm link during development:
```bash
cd claude-code-files-parser && npm link
cd ../karma-logger && npm link claude-code-files-parser
```

## Risk Assessment

| Risk | Mitigation |
|------|------------|
| Breaking karma-logger imports | Automated find/replace, test coverage |
| Type mismatches after split | Export identical types, run tsc |
| Missing exports | Start with `index.ts` re-exporting everything |
| Circular dependencies | Parser has no deps on karma-logger |

## Success Criteria

1. `claude-code-files-parser` builds independently
2. karma-logger imports from new package with no code changes
3. All existing karma-logger tests pass
4. Parser has its own test suite with fixtures
