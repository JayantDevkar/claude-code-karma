## Open Issues

### [OPEN] State Inference False "Failed" Detection

**Severity**: Medium
**Impact**: Active/completed agents incorrectly shown as "failed"

**Observation** (from meta-testing 2026-01-09):
```
Actual task status: completed (all 4 doc subagents finished successfully)
Radio scan status:  2 completed, 2 failed (a19e094, ae635d5)
```

The state inference in `subagent-watcher.ts` marks agents as "failed" when:
- Any `tool_result` has `is_error: true` (even recoverable errors like file not found)
- The agent was actively working but encountered a single tool error during exploration

**Root Cause**: The `hasToolError` flag triggers "failed" state without considering:
1. Whether the error was fatal or recoverable
2. Whether the agent continued working after the error
3. The actual final stop_reason

**Potential Fix**: Only mark as "failed" when:
- `stop_reason === 'error'` (explicit error termination)
- Or file is stale AND last message has `is_error: true`
- Ignore transient tool errors if agent continues working

## Resolved Issues

### [FIXED] Claude Code Task Tool - No KARMA_* Environment Variables

**Severity**: High
**Impact**: Subagents spawned via Task tool don't register in real-time radio

**Observation** (from meta-testing 2026-01-09):
```
Radio agents (real-time IPC): 2
Subagents (JSONL scan):       8
```

Claude Code's Task tool spawns subagents without injecting `KARMA_*` environment variables:
- `KARMA_AGENT_ID` - not set
- `KARMA_SESSION_ID` - not set
- `KARMA_PARENT_ID` - not set

**Root Cause**: Task tool is internal to Claude Code and doesn't know about karma-logger's IPC protocol.

**Solution Implemented**: Auto-start subagent watcher in dashboard server (`src/dashboard/server.ts`)
- When radio is enabled and sessionId is provided, dashboard now auto-starts subagent watcher
- Bridges JSONL files to Radio with 2s polling interval
- Cleanup handled on server stop

### [FIXED] TTL Mismatch - Session List vs Agent Status

**Severity**: Low
**Impact**: Stale agent IDs in session list after status expires

**Observation**:
```typescript
const TTL = {
  STATUS: 300000,           // 5 minutes
  SESSION_AGENTS: 3600000,  // 1 hour (12x longer)
};
```

Agent status can expire while ID remains in session list. Current code handles this by filtering nulls, but creates inconsistent counts.

**Solution**: Already implemented in `src/walkie-talkie/agent-radio.ts`
- `refreshSessionTTL()` method is called on every `setStatus()` update
- Keeps SESSION_AGENTS and parent children list TTLs refreshed while agents are active

### [FIXED] Warmup Agents Pollute Agent List

**Severity**: Low
**Impact**: Internal warmup agents appear in user-facing agent lists

**Observation** (from meta-testing):
```
a1049fa    failed     opus-4-5-20251101    157    Warmup
a5176a3    completed  haiku-4-5-20251001   0      Warmup
aad9478    completed  opus-4-5-20251101    0      Warmup
```

Claude Code's internal warmup agents are detected and reported alongside user-spawned agents.

**Solution Implemented**: Filter in `src/walkie-talkie/subagent-watcher.ts` (`parseSubagentJsonl`)
- Filters agents where task === "Warmup" (case-insensitive)
- Filters agents with toolCount === 0 AND no assistant messages
- Returns null to exclude from agent list

### [FIXED] No Subagent Depth Tracking

**Severity**: Low
**Impact**: Can't distinguish nested subagent levels

**Observation**: JSONL files are flat - no indication if a subagent spawned another subagent.

The `parentId` in radio comes from the watching session, not the actual spawning agent. This means:
- All subagents appear as direct children of the main session
- True multi-level hierarchies (agent A spawns agent B spawns agent C) not captured

**Root Cause**: JSONL files don't record who spawned them, only the root session ID.

**Solution Implemented**: Depth field added to AgentStatus
- `depth` field in `AgentStatus` type (`src/walkie-talkie/types.ts:122`)
- Computed in `AgentRadioImpl` constructor (`src/walkie-talkie/agent-radio.ts`)
  - Session children: depth = 0
  - Agent children: depth = parent.depth + 1
- Added to `SubagentInfo` in subagent-watcher (default: 0 for JSONL-discovered agents)

**Limitation**: JSONL-discovered agents always have depth=0. True depth tracking requires agents to register via radio IPC.
