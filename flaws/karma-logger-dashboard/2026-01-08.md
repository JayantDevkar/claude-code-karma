## Identified Gaps

### 1. Missing CLI Command: `karma status --tree`

**Severity:** Medium
**Plan Reference:** DASHBOARD_MVP_PLAN.md, line 102-103

**Expected:**
```bash
karma status --tree       # Agent hierarchy tree
```

**Actual:**
The `--tree` flag is not implemented in the status command.

**Current status command options:**
```typescript
// src/cli.ts:36-60
.option('-p, --project <name>', 'Show status for specific project')
.option('-a, --all', 'Show all active sessions')
.option('-j, --json', 'Output as JSON')
// Missing: .option('-t, --tree', 'Display agent hierarchy tree')
```

**Impact:**
Users cannot view agent hierarchy from the status command. The `AgentTree.tsx` component exists but is only accessible via `karma watch --ui`.

**Recommended Fix:**
```typescript
// Add to status command options
.option('-t, --tree', 'Display agent hierarchy tree')

// In command handler, render tree using existing AgentTree logic
if (options.tree) {
  // Reuse useAgentTree.ts logic for CLI output
}
```

### 2. Missing Separate `charts.js` File

**Severity:** Low
**Plan Reference:** DASHBOARD_MVP_PLAN.md, line 256

**Expected Structure:**
```
src/dashboard/public/
├── index.html
├── style.css
├── app.js          # Petite-Vue app
└── charts.js       # uPlot setup (MISSING)
```

**Actual Structure:**
```
src/dashboard/public/
├── index.html
├── style.css
└── app.js          # Contains both Petite-Vue AND uPlot logic
```

**Impact:**
Minor structural deviation. Chart logic is embedded in `app.js` rather than separated. This affects maintainability but not functionality.

**Recommendation:**
Consider extracting chart-related code to `charts.js` for better separation of concerns. Low priority.

---

### 3. Chart Data Retention Not Configured

**Severity:** Low
**Plan Reference:** DASHBOARD_MVP_PLAN.md, line 319

**Plan Recommendation:**
> How many data points to keep in memory?
> *Recommendation*: 3600 points (1 hour at 1Hz)

**Actual:**
Data retention limit not explicitly configured. The `useTokenFlow.ts` hook uses a buffer but the size is not documented or easily configurable.

**Recommendation:**
Add configurable constant for chart data retention:
```typescript
const CHART_DATA_POINTS = 3600; // 1 hour at 1Hz
```
